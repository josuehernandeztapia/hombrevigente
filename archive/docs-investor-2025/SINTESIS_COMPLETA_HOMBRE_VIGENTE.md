### Data Flywheel: El Foso Competitivo

**Arquitectura Event-Driven (EDA):**

```
[Fuentes Externas] → [Event Bus: Kafka/Pub-Sub] → [Streaming: Dataflow]
     ↓                                                       ↓
[Booksy, Stripe]                                   [Data Lakehouse: BigQuery]
[Imaging Module]                                            ↓
[IoT Sensors]                                          [dbt transforms]
                                                             ↓
                                                        [gold tables]
                                                             ↓
                                                    [Feed → 9 AI Agents]
                                                             ↓
                                                    [Actions → Event Bus]
```

**El Ciclo Autorreforzado:**
1. Captura de eventos (diagnósticos, citas, pagos, IoT)
2. Procesamiento en streaming (limpieza, enriquecimiento)
3. Almacenamiento en Lakehouse (SSOT)
4. Consumo por agentes AI (análisis, predicciones)
5. Acciones automatizadas (ofertas, ajustes pricing)
6. Generación de nuevos eventos → **loop infinito que mejora**

**Resultado:** Mientras más clientes, más datos, mejores predicciones, mayor eficiencia, mayor rentabilidad, más capital para expansión.

## 🚦 RISKGUARD AI: GOBERNANZA FINANCIERA EN TIEMPO REAL

### El Semáforo Homeostático

**Verde (Seguro):**
- Altman Z-Score >2.9
- Probabilidad quiebra <5%
- OptiVigente AI opera sin restricciones

**Amarillo (Precaución):**
- Z-Score 1.8-2.9
- P(quiebra) 5-10%
- RiskGuard AI limita descuentos (protege márgenes)

**Rojo (Alerta Crítica):**
- Z-Score <1.8
- P(quiebra) >10%
- **Bloqueo automático** de nuevos descuentos
- Alertas a CFO/COO vía Slack

---

### Fórmula Altman Z-Score (adaptada para servicios)

```python
Z = 1.2×(Working Capital/Total Assets)
  + 1.4×(Retained Earnings/Total Assets)
  + 3.3×(EBIT/Total Assets)
  + 0.6×(Market Value Equity/Book Value Liabilities)
  + 1.0×(Sales/Total Assets)
```

**Interpretación:**
- Z >2.9: Zona segura
- 1.8 < Z < 2.9: Zona gris
- Z <1.8: Alto riesgo

### 2. El Data Flywheel (Propiedad Intelectual)

**La tesis técnica:**

El modelo Hombre Vigente NO es replicable por competidores porque:

1. **Disciplina Minimalista** es prerrequisito para OptiVigente AI (MILP solo funciona con catálogo limitado)
2. **Tecnología Clínica** genera datos estructurados propietarios (Imaging Module)
3. **Comunidad CLUB** genera engagement recurrente = más datos = mejores modelos
4. **Resultado:** Entre más clientes, más difícil de copiar (network effects + data moat)

**Fórmula del foso:**
```
Competencia tradicional: Mejor servicio ≠ Mejor data ≠ Mejor IA
Hombre Vigente: Mejor servicio = Mejor data = Mejor IA = Mejor servicio
```
